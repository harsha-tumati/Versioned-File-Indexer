# Memory-Efficient Versioned File Indexer Using C++

## Overview

This project implements a text dataset analyzer that processes large text files efficiently and supports multiple queries on the words contained in the dataset. Instead of loading the entire file into memory, the program reads the file in fixed-size chunks and processes it incrementally. This approach allows the system to handle very large datasets while keeping memory usage low.

The program extracts alphanumeric words from the input file, converts them to lowercase, and stores their frequencies in a versioned index. Each dataset is associated with a version name so that comparisons can be made between different versions of the dataset.

The system supports three main types of queries:

* Word Query – Returns the number of occurrences of a specific word in a given dataset version.
* Top-K Query – Returns the top k most frequent words in a dataset version.
* Diff Query – Compares the frequency of a word between two versions of the dataset.

The implementation uses an object-oriented design where different classes handle file reading, tokenization, indexing, and query execution.

## Key Features

* Buffered file processing using a fixed-size buffer
* Efficient word extraction using a tokenizer
* Handling words split across buffer boundaries
* Fast indexing using hash maps
* Efficient Top-K query using a min-heap
* Object-oriented design with inheritance and polymorphism

## Buffered File Processing

The program processes the input file using a fixed-size buffer instead of reading the entire file at once.

Internally, file reading is performed using the C++ function:

ifstream.read()

The file stream maintains an internal pointer that keeps track of the current reading position. Each call to read() reads data starting from the current pointer location and then moves the pointer forward by the number of bytes read.

Because of this behavior, repeated calls to read() automatically read the next chunk of the file rather than reading the same data again.

The program also uses the gcount() function to determine how many bytes were actually read from the file.

This chunk-based approach ensures that the program remains memory efficient even when processing very large files.

---

## Tokenization

The tokenizer extracts valid words from the stream of characters.

A word is defined as a sequence of alphanumeric characters. Any character that is not alphanumeric acts as a delimiter.

Since the file is processed in chunks, a word may sometimes be split across two adjacent buffers. To handle this situation, the tokenizer uses a temporary variable called carry_.

If the tokenizer reaches the end of a chunk while scanning a word, the partial word is stored in carry_. When the next chunk is processed, the stored fragment is attached to the beginning of the new chunk so the complete word can be reconstructed.

All words are converted to lowercase to ensure case-insensitive comparisons.

## Versioned Word Index

Word frequencies are stored using a versioned index structure implemented with unordered_map.

The structure is:

version → (word → count)

The outer unordered_map stores dataset versions, while each version maps to another unordered_map that stores the frequency of each word.

Example structure:

v1 → { word → count }
v2 → { word → count }

The unordered_map data structure uses hashing, which provides average O(1) insertion and lookup time. This makes the indexing process efficient even when the dataset contains a very large number of words.

---

## Top-K Query Optimization

The Top-K query returns the most frequent k words from the dataset.

Instead of sorting all words, the program uses a priority queue implemented as a min-heap.

The algorithm works as follows:

1. Iterate through all word-frequency pairs.
2. Insert each pair into the heap.
3. If the heap size exceeds k, remove the smallest element.

This ensures that the heap always contains only the k largest frequency values.

The time complexity of this approach is:

O(n log k)

which is more efficient than sorting the entire dataset.

## Object-Oriented Design

The program uses several object-oriented programming concepts.

Inheritance is used in the Reader hierarchy where BufferedFileReader extends the Reader interface.

Polymorphism is used in the Query hierarchy where different query classes implement the execute() function.

Encapsulation is used in the VersionedIndex class which manages all operations related to storing and retrieving word frequencies.

This modular design separates responsibilities across different classes and makes the system easier to extend and maintain.

## Compilation

Compile the program using the following command:


```bash
g++ -std=c++17 analyzer.cpp -o analyzer
```
---

## Usage Examples

Word Query

```bash
./analyzer --file dataset_v1.txt --version v1 --buffer 512 --query word --word error
```

### Top-K Query

```bash
./analyzer --file dataset_v1.txt --version v1 --buffer 512 --query top --top 10
```

### Diff Query

```bash
./analyzer --file1 dataset_v1.txt --version1 v1 --file2 dataset_v2.txt --version2 v2 --buffer 512 --query diff --word error
```
---

## Command Line Arguments

```
--file : Input dataset file
--file1 : First dataset file (for diff queries)
--file2 : Second dataset file (for diff queries)
--version : Version name of dataset
--version1 : Version name of first dataset
--version2 : Version name of second dataset
--buffer : Buffer size in KB (256–1024)
--query : Query type (word, top, diff)
--word : Word used in queries
--top : Number of top words to return
```

---

## Assumptions

* Words consist only of alphanumeric characters.
* Word comparisons are case-insensitive.
* Buffer size must be between 256 KB and 1024 KB.
* Input files must exist and be accessible.
